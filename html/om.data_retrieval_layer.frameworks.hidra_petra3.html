The hidra_petra3 Module¶

Retrieval of events from HiDRA at Petra III.

This module contains functions and classes that retrieve data events from the HiDRA framework at the PETRA III facility.

initialize_event_source(source, node_pool_size, monitor_params)[source]¶

Initializes the HiDRA event source at Petra III.

This function must be called on the master node before the event_generator() function is called on the worker nodes.

Parameters

  • source (str) – the hostname or ip address of the machine where HiDRA is running.

  • node_pool_size (int) – the total number of nodes in the Om pool, including all the worker nodes and the master node.

  • monitor_params (MonitorParams) – an object storing the Om monitor parameters from the configuration file.

Return type

Any

Returns

Dict[str, Any] a dictionary storing the HiDRA initialization information.

Raises

OndaHidraAPIError – if the initial connection to HiDRA fails.

event_generator(source, node_rank, node_pool_size, monitor_params)[source]¶

Retrieves events to process from HiDRA at Petra III.

This function must be called on each worker node after the initialize_event_source() function has been called on the master node. The function is a generator and it returns an iterator over the events that the calling worker must process.

Parameters

  • source (str) – the hostname or ip address of the machine where HiDRA is running.

  • node_rank (int) – the rank, in the OM pool, of the worker node calling the function.

  • node_pool_size (int) – the total number of nodes in the OM pool, including all the worker nodes and the master node.

  • monitor_params (MonitorParams) – an object storing the OM monitor parameters from the configuration file.

Yields

DataEvent – an object storing the event data.

Raises

OndaHidraAPIError – if the initial connection to HiDRA fails.

Return type

Generator[DataEvent, None, None]

timestamp(event)[source]¶

Gets the timestamp of an event retrieved from HiDRA at Petra III

A HiDRA event usually corresponds to a single data file written by a detector. The creation date and time of the file is used as timestamp for the event.

Parameters

event (DataEvent) – an object storing the event data.

Returns

the timestamp of the event in seconds from the Epoch.

Return type

numpy.float64

beam_energy(event)[source]¶

Gets the beam energy for an event retrieved from HiDRA at Petra III.

HiDRA events do not usually contain beam energy information. This function takes the beam energy value from the configuration file, specifically from the ‘fallback_beam_energy_in_eV’ entry in the ‘DataRetrievalLayer’ parameter group.

Parameters

event (DataEvent) – an object storing the event data.

Returns

the energy of the beam in eV.

Return type

float

detector_distance(event)[source]¶

Gets the detector distance for an event retrieved from HiDRA at Petra III.

HiDRA events don’t usually contain detector distance information. This function takes it from the configuration file, specifically from the ‘fallback_detector_distance_in_mm’ entry in the ‘DataRetrievalLayer’ parameter group.

Parameters

event (DataEvent) – an object storing the event data.

Returns

the detector distance in mm.

Return type

float
