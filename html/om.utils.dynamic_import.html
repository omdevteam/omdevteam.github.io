The dynamic_import Module¶

Dynamic import from various OM layers.

This module contains functions that import required modules, classes and functions from different parts of OM.

import_processing_layer(processing_layer_filename)[source]¶

Imports the specified Processing Layer.

This function searches for the python file with the implementation of the Processing Layer in the working directory first. If the file is not found there, this function looks for it in the standard OM folder structure.

Parameters

processing_layer_filename (str) – the name of a python file containing the processing layer implementation.

Returns

the imported processing layer.

Return type

ModuleType

get_event_handling_funcs(data_retrieval_layer)[source]¶

Retrieves Event Handling Functions from the Data Retrieval Layer.

This function retrieves the following Event Handling functions:

  • ‘initialize_event_source’

  • ‘event_generator’

  • ‘open_even’

  • ‘close_event’

  • ‘get_num_frames_in_event’

Parameters

data_retrieval_layer (ModuleType) – the Data Retrieval Layer module.

Returns

a dictionary containing the Event Handling functions:

  • ’initialize_event_source’ [[str, int, MonitorParams], Any]

  • ’event_generator’ [[str, int, int, parameters.MonitorParams], Generator[DataEvent, None, None]]

  • ’open_event’ [[DataEvent], None]

  • ’close_event’ [[DataEvent], None],

  • ’get_num_frames_in_event’ [[DataEvent], int]

  • The dictionary contains keys corresponding to all these function names.

  • The corresponding dictionary values store the function implementations.

Return type

Dict

Raises

OmMissingEventHandlingFunctionError – if one of the Event Handling Function is not found in the Data Retrieval Layer.

get_data_extraction_funcs(required_data, data_retrieval_layer)[source]¶

Retrieves Data Extraction Functions from the Data Retrieval Layer.

Parameters

  • required_data (List[str]) – a list with the the names of the Data Extraction Functions that must be retrieved.

  • data_retrieval_layer (TypeModule) – the Data Retrieval Layer module.

Returns

a dictionary whose keys match the names in the ‘required_data’ argument, and whose values store the corresponding function implementations.

Return type

Dict[str, Callable[[data_event.DataEvent], Any]]

Raises

OmMissingDataExtractionFunctionError – if a required Data Extraction Function is not found in the Data Retieval Layer.

get_psana_detector_interface_funcs(required_data, data_retrieval_layer)[source]¶

Retrieves the psana Detector Interface Initialization Functions.

Parameters

  • required_data (List[str]) – a list with the names of the psana Detector Interface Initialization Functions that must be retrieved.

  • data_retrieval_layer (TypeModule) – the Data Retrieval Layer module.

Returns

a dictionary whose keys match the names in the ‘required_data’ argument, but with an ‘_init’ extension appended at the end of each name. The corresponding dictionary values store the function implementations.

Return type

Dict[str, Callable[[str], Any]]

Raises

MissingPsanaInitializationFunctionError – if a required psana Detector Interface Initialization Function is not found in the Data Retrieval layer.
